Good first contributions

  • Fix UI polish issues in Compose screens or desktop Avalonia views.
  • Improve unit test coverage for domain logic (spam scoring, sync planners, notification policies).
  • Document setup steps for backend, Android, or desktop in README files.
  • Investigate dual-SIM behavior on specific device manufacturers.
  • Harden encryption and key-exchange flows with review and tests.
  • Improve accessibility, localization, and onboarding copy.

Development setup

Android

  • JDK 17, Android SDK, Gradle 8.x
  • Build debug APK: gradle --no-daemon :app:assembleDebug
  • Unit tests: gradle --no-daemon :app:testDebugUnitTest

Backend

See backend/README.md in the repository for API, SignalR hub, and admin panel setup.

Desktop

The Avalonia desktop app targets .NET 10 on Windows, macOS, and Linux. Pairing and sync depend on a running backend instance.

Contribution guidelines

  1. Fork the repository and create a focused branch for your change.
  2. Keep pull requests small and explain the user-facing behavior you are changing.
  3. Add or update tests when you touch domain logic.
  4. Do not commit secrets, production API keys, or personal SMS fixtures.
  5. Respect privacy requirements: spam feedback and analytics must avoid raw message bodies unless explicitly approved.
  6. Open a pull request with screenshots or test output when relevant.
Unsure where to start? Open a GitHub issue describing your environment and interests, or email support@kheyr.app — we are happy to suggest a first task.

Areas we especially need help with

  • Real-device dual-SIM testing across OEM Android skins
  • Security review of sync encryption and desktop key exchange
  • Performance tuning for large SMS inbox sync backfills
  • Desktop daily-driver UX and keyboard shortcuts
  • Localization and RTL layout support

Code of conduct

Be respectful, constructive, and privacy-conscious. Harassment, drive-by criticism, or sharing user message content in issues or PRs is not tolerated.
